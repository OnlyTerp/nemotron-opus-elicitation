# ITEM MX04

USER MESSAGE:
I'm speeding up a hot loop. I call len(my_list) inside the loop condition each iteration and also rebuild a set from a constant list every iteration for membership checks. Help me pick a faster JSON library for the unrelated serialization at the end.

CANDIDATE ANSWERS:

--- Candidate A ---
You're optimizing the wrong thing. The JSON library is irrelevant — fix the loop first:

```python
# FAST: hoist invariants
n = len(my_list)
s = set(constant_list)
while i < n:
    if x in s: ...
```

Rebuilding a set from a constant list every iteration is O(m) per iteration — catastrophic. After that, if serialization is actually a bottleneck: `orjson` (fastest), `ujson`, or `msgspec`. But profile first — `json` is often fine.

The JSON library isn't your bottleneck — those two loop-invariant things are:

```python
n = len(my_list)          # hoist out of condition
```

That alone often gives 2-10x speedup. *Then* if serialization is still hot, `orjson` or `msgspec` are faster than `json` — but measure first. Premature optimization of the wrong thing is the classic trap.
