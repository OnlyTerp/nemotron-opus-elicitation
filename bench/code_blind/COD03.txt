# ITEM COD03

USER MESSAGE:
Naming nit: is `getData` a good name or should it be more specific? Reviewing a teammate's PR.
```js
async function getData(urls) {
  const results = [];
  urls.forEach(async (u) => {
    const r = await fetch(u);
    results.push(await r.json());
  });
  return results;
}
```

CANDIDATE ANSWERS:

--- Candidate A ---
`getData` is vague — what data? From where? Name it for what it returns: `fetchJsonFromUrls` or `loadResources`. Also, the code is broken: `forEach` doesn't await the async callbacks, so it returns an empty array. Use `Promise.all(urls.map(u => fetch(u).then(r => r.json())))`. The name is the least of it.

--- Candidate B ---
`getData` is too vague — it tells me *that* you're getting data, not *what* data or *from where*. Since it fetches JSON from URLs, name it for what it actually returns: `fetchJsonFromUrls`, `loadJsonEndpoints`, or `getJsonResponses`. The current name forces readers to open the function to understand its purpose. Also, the `forEach` + `async` pattern is broken — it doesn't await the promises, so `results` returns empty. Use `Promise.all(urls.map(u => fetch(u).then(r => r.json())))` instead.

--- Candidate C ---
`getData` is too generic — it doesn't convey *what* data, *from where*, or *what shape* it returns. Prefer something like `fetchJsonFromUrls` or `getUserProfiles` depending on the actual use case.

Also, there's a **bug**: `forEach` doesn't await async callbacks. The function returns `[]` immediately. Fix with `for...of` or `Promise.all`:

```js
async function fetchJsonFromUrls(urls) {
  const results = [];
  for (const u of urls) {
    const r = await fetch(u);
    results.push(await r.json());
  }
  return results;
}

// Or parallel (faster):
async function fetchJsonFromUrls(urls) {
  return Promise.all(urls.map(u => fetch(u).then(r => r.json())));
}
```

The parallel version is usually preferable unless you need sequential ordering or rate limiting.

--- Candidate H ---
`getData` is terrible—it says nothing about *what* data, *from where*, or *how*. It fetches JSON from URLs, so name it `fetchJsonFromUrls`.

**Critical bug:** `forEach` doesn't await the async callbacks. The function returns `[]` before any fetch completes. Use `Promise.all` with `map`:

```js
async function fetchJsonFromUrls(urls) {
  return Promise.all(urls.map(async u => (await fetch(u)).json()));
}
```

This fixes the bug, names the operation precisely, and is shorter.
