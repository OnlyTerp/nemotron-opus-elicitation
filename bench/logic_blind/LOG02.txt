# ITEM LOG02

USER MESSAGE:
Is `acc` a clear enough parameter name here, or should I spell it out?
```python
def collect(item, acc=[]):
    acc.append(item)
    return acc
```

CANDIDATE ANSWERS:

--- Candidate A ---
Spell it out: `accumulator` or `result`. `acc` is a common abbreviation but still imposes a tiny cognitive tax on readers unfamiliar with the convention. More critically, this function has a **bug**: the mutable default `acc=[]` is created once at definition time, so repeated calls without an argument share the same list. Fix that first — use `acc=None` and initialize inside — then name the parameter clearly.

--- Candidate B ---
`acc` is fine — it's the standard abbreviation for "accumulator" in Python (fold/reduce contexts). Anyone reading this pattern will recognize it instantly.

The real problem: `acc=[]` as a default argument is a bug. The same list object persists across calls:

```python
collect(1)  # [1]
collect(2)  # [1, 2] — oops
```

Fix it:

```python
def collect(item, acc=None):
    if acc is None:
        acc = []
    acc.append(item)
    return acc
```

Or better, don't mutate — return a new list:

```python
def collect(item, acc=None):
    acc = [] if acc is None else acc
    return acc + [item]
```

If you want in-place mutation, make the caller pass the list explicitly.

--- Candidate D ---
Two issues here: the name and the mutable default.

`acc` is too cryptic — call it `accumulator` or `collected`. But the real problem is `acc=[]` as a default. In Python, that list is created once at function definition time and shared across all calls without an explicit argument. This is almost certainly a bug.

Fix both:
```python

    if accumulator is None:
        accumulator = []

    return accumulator
```

Now the name is clear and each call gets its own fresh list unless you explicitly pass one.
