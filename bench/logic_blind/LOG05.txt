USER MESSAGE:
Rename `g` to `grid`, or is `g` fine in a 4-line function?
```python
def make_grid(rows, cols):
    g = [[0] * cols] * rows
    return g
```

CANDIDATE ANSWERS:

--- Candidate A ---
Rename to `grid`. The bug is line 11: `[[0] * cols] * rows` creates `rows` references to the *same* inner list. Mutating `grid[0][0] = 1` changes every row. Fix:
```python
def make_grid(rows, cols):
    grid = [[0] * cols for _ in range(rows)]
    return grid
```
With the fix, `grid` is self-documenting; `g` forces the reader to trace the bug.
