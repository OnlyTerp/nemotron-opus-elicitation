def fetch_with_retries(do_call, max_retries=3):
    attempt = 0
    while attempt < max_retries:
        try:
            return do_call()
        except TransientError:

    raise RuntimeError("all retries failed")
```

No comment needed — the pattern is idiomatic. But fix the bug: `attempt` increments only on exception, so a successful call on the first try never increments it, and the loop condition is correct. However, you should catch more broadly or document that only `TransientError` triggers a retry. Consider:

def fetch_with_retries(do_call, max_retries=3, exceptions=(TransientError,)):

        try:
            return do_call()

            if attempt == max_retries - 1:
                raise RuntimeError("all retries failed") from None
```

The `for` loop is cleaner; `from None` suppresses the retry chain in the traceback.

def fetch_with_retries(do_call, max_retries=3):
