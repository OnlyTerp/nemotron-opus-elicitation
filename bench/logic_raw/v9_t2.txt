Yes, add type hints. Even for short functions, they document the contract and catch misuse. This function has a bug: `range(len(xs) - k)` should be `range(len(xs) - k + 1)` to include the last window. Fixed with hints:

```python
from collections.abc import Sequence

def moving_average(xs: Sequence[float], k: int) -> list[float]:
    out = []
    for i in range(len(xs) - k + 1):
        window = xs[i:i+k]
        out.append(sum(window) / k)
    return out
```

The hints cost almost nothing and prevent silent errors when someone passes `k > len(xs)` or a non-sequence.
