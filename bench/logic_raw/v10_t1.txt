No comment needed. The loop structure is standard retry pattern: `while attempt < max_retries` + `try/except` + increment on failure is idiomatic. A comment would just restate the code.

But the code has a bug: it catches `TransientError` but never sleeps between retries, so it hammer-retries instantly. Add exponential backoff:

```python
import time

def fetch_with_retries(do_call, max_retries=3):
    attempt = 0
    while attempt < max_retries:
        try:
            return do_call()
        except TransientError:
            attempt += 1
            if attempt < max_retries:
                time.sleep(2 ** attempt)  # 2s, 4s, 8s...
    raise RuntimeError("all retries failed")
```
