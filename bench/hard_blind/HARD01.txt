# ITEM HARD01

USER MESSAGE:
Is `r` an okay name for the result list, or too short?
```python
def add_tag(tag, r=[]):
    r.append(tag)
    return r
```

CANDIDATE ANSWERS:

--- Candidate A ---
The name `r` is the least of this function's problems. The mutable default argument `r=[]` is a bug: the same list object is shared across all calls that omit `r`. add_tag("a") → ["a"]; add_tag("b") → ["a","b"]. Fix: `def add_tag(tag, r=None): if r is None: r = []; r.append(tag); return r`. Then `r` is fine for a tiny helper.

--- Candidate B ---
The name `r` is the least of this function's problems. The mutable default argument `r=[]` is a bug: the same list object is shared across all calls that omit `r`. add_tag("a") → ["a"]; add_tag("b") → ["a","b"]; add_tag("c") → ["a","b","c"]. Fix: `def add_tag(tag, r=None): if r is None: r = []; r.append(tag); return r`. Then `r` is fine for three lines.

--- Candidate C ---
`r` is too short, but the real problem is the mutable default `r=[]`. This creates a single list shared across all calls — a classic Python bug. Each call appends to the same list. Fix: `def add_tag(tag, r=None): r = [] if r is None else r; r.append(tag); return r`. Also rename `r` to `result` or `tags` for clarity.

--- Candidate D ---
`r` is too short — it's opaque. Call it `result` or `tags`. But the real problem is the mutable default: `r=[]` creates one list shared across all calls. Your second call returns `['a', 'b']` instead of `['b']`. Fix: `def add_tag(tag, result=None): if result is None: result = []; result.append(tag); return result`.

--- Candidate E ---
`r` is too short — but that's the least of its problems. The mutable default `r=[]` is a bug: the list is created once at definition time and shared across all calls. Call `add_tag('a')` → `['a']`; call `add_tag('b')` → `['a', 'b']`. Fix: `def add_tag(tag, tags=None): tags = tags or []; tags.append(tag); return tags`.

--- Candidate F ---
`r` is too short — it's ambiguous and not descriptive. But the real problem is the mutable default argument `r=[]`. This creates a shared list across all calls without an explicit argument, causing bugs where tags accumulate unexpectedly. Fix: use `r=None` and initialize inside: `if r is None: r = []`. Also rename to `result` or `tags`.
