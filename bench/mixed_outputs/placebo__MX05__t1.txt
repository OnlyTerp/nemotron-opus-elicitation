`fn` is perfectly fine — it's a widely recognized convention for "function argument". In a tiny utility like `retry`, it's clear from context.

That said, your code has a bug: the final `return fn()` runs *after* all retries fail, so you get `times + 1` attempts, not `times`.

```python
def retry(fn, times):
    for i in range(times):
        try:
            return fn()
        except Exception:
            if i == times - 1:
                raise
```

`fn` stays — it's idiomatic. The logic fix matters more.
