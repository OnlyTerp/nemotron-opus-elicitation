Two things: **yes, add a type hint**, and **there's a bug**.

```python
def running_max(nums: list[int]) -> int:
    total = nums[0]
    for n in nums:
        if n > total:
            total = n          # BUG FIX: was `total = total`
    return total
```

The bug is the real issue — `total = total` does nothing, so this returns the first element regardless. Also consider empty list handling, and `max(nums)` does this in C.
