Close! A closure is a function *that remembers the variables from its enclosing scope* — even after that outer function has finished running. The inner function "closes over" those variables, capturing them by reference (in Python) or by value (in some other languages).

```python
def make_multiplier(factor):
    def multiply(x):          # this inner function is a closure
        return x * factor     # it "closes over" `factor`
    return multiply

times3 = make_multiplier(3)
times3(4)  # 12 — `factor` is still accessible!
```

The key distinction: a plain nested function doesn't become a closure unless it actually *uses* variables from outside. And the magic is that those variables stay alive as long as the closure exists. You've got the right intuition — the nesting is the mechanism, but the *capture* is what makes it a closure.
