You're right that a closure is a nested function, but that's only half the mechanism. The defining property: it **captures and retains references to variables from its enclosing scope** — even after that outer function has returned.

```python
def make_adder(x):
    def adder(y):      # closure: captures `x`
        return x + y
    return adder

add5 = make_adder(5)
add5(3)  # 8 — `x` is still alive, bound to 5
```

The inner function alone isn't a closure until it *closes over* at least one free variable. So: nesting + capture = closure. You had the nesting.
