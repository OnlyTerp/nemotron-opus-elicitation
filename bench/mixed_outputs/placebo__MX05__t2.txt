`fn` is fine — a common convention for callbacks. But your code has a bug: after the loop exhausts, the final `return fn()` runs one more unguarded attempt (times=3 → 4 attempts, raw exception propagates).

```python
def retry(fn, times):
    for i in range(times):
        try:
            return fn()
        except Exception:
            if i == times - 1:
                raise
```
