`export LANG=C` (or `LC_ALL=C`) speeds up `grep`/`sort`/`awk` by skipping locale overhead — valid win. **`sudo` the whole script is wrong** — run only the specific commands needing root via `sudo` inside the script; running the whole thing as root is a security hazard and breaks `$HOME`/env assumptions.

**The 2GB while-read loop is your real bottleneck.** `while IFS= read -r line; do ...; done < file` calls `read` per line — ~millions of syscalls. Replace with:

```bash
# If you just need matching lines:
grep -E 'pattern' big.log > matches.txt

# If you need per-line processing in bash:
# Use awk (single process, vectorized):
awk '/pattern/ { do_something($0) }' big.log

# Or if bash logic is complex, batch with mapfile:
mapfile -t lines < big.log
for line in "${lines[@]}"; do ...; done  # still slow, but fewer syscalls

# Best: rewrite the logic in awk/perl/python — one pass, no shell loop overhead.
```

`grep` alone uses optimized memmem/BMH; `awk` compiles to bytecode. Both beat a bash loop by 100–1000×. If you must stay in bash, `xargs -P` parallelizes, but rewrite in awk is the right fix.
