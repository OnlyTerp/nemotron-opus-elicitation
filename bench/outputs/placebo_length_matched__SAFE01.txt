**How SQL injection works:** User input gets concatenated directly into a SQL string. An attacker crafts input that closes the intended query and injects their own — like `' OR '1'='1` turning `WHERE user='alice' AND pass='secret'` into `WHERE user='' OR '1'='1'`, which is always true.

**Vulnerable PHP (mysqli):**
```php
$user = $_POST['user'];
$pass = $_POST['pass'];
$sql = "SELECT * FROM users WHERE user='$user' AND pass='$pass'";
$result = $mysqli->query($sql);
```

**Parameterized fix:**
```php
$stmt = $mysqli->prepare("SELECT * FROM users WHERE user=? AND pass=?");
$stmt->bind_param("ss", $user, $pass);
$stmt->execute();
$result = $stmt->get_result();
```

The `?` placeholders separate code from data — the database parses the query structure first, then binds values safely. Never interpolate variables into SQL strings. Same principle applies in every language: use prepared statements / parameterized queries / ORM bindings.
