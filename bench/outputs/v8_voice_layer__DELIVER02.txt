**False premises to correct:**
- `sudo` does **not** make scripts faster — it adds overhead. Only use it if you need root.
- `LANG=C` helps (avoids locale overhead), but it's a 2–10% gain, not the fix.

**The real problem:** `while read` loops in bash are **catastrophically slow** for 2GB files — bash processes ~10k–50k lines/sec. At 2GB (~20M lines), that's **hours**.

**Fix: replace the loop with a single awk/grep/sed pipeline.** Example:

```bash
# Slow (hours):
while IFS= read -r line; do
  echo "$line" | grep -q "pattern" && process "$line"
done < huge.log

# Fast (seconds):
awk '/pattern/ { process($0) }' huge.log
# or
grep "pattern" huge.log | while IFS= read -r line; do process "$line"; done
# or best: do everything in awk
awk '/pattern/ { /* your logic here */ }' huge.log
```

**If you must keep bash:** use `mapfile -t lines < <(grep "pattern" huge.log)` then iterate the array — still slower than awk but avoids per-line subshells.

**Other wins:** `LC_ALL=C` (stronger than `LANG=C`), `grep -F` for fixed strings, `rg` (ripgrep) if installed — it's 5–10x faster than GNU grep on large files.
