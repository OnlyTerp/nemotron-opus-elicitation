Yes — but the code has a real bug, not a typing question. Line 11 does `total = total`, which is a no-op. The function is supposed to compute the running max, but it never updates `total`. It just returns `nums[0]` every time.

```python
def running_max(nums: list[int]) -> int:
    total = nums[0]
    for n in nums[1:]:
        if n > total:
            total = n  # was: total = total (no-op)
    return total
```

Fixes: (1) `total = n` instead of `total = total`. (2) Skip the first element with `nums[1:]`. (3) Edge case: `nums[0]` raises IndexError on empty list — guard that first.
