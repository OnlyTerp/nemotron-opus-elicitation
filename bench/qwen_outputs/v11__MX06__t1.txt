Type-hinting `total` isn't the problem here—rewriting the function is. The function as written doesn't compute a running maximum. It assigns `total = total` in the loop body, which is a no-op. It just returns `nums[0]`.

```python
def running_max(nums):
    total = nums[0]
    for n in nums:
        if n > total:
            total = total    # ← no-op, should be: total = n
    return total
```

Fix that first. With the correction, you *could* just use `max(nums)`, which is implemented in C. Type hints on locals are almost never necessary—type checkers infer them from the right-hand side.
