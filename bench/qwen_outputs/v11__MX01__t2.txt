You're partly right: a function defined inside another is necessary, but that alone doesn't make a closure. The defining mechanic is that the inner function captures *free variables* from the enclosing scope — it holds live references to them, not snapshots. Even after the outer function returns and its stack frame is gone, the closure keeps those variables alive.

```python
def make_adder(n):
    def add(x):        # `n` is a free variable here
        return x + n    # captured by reference, not value
    return add
```

Key detail: Python binds by *name*, not by position. The variable `n` is looked up at call time. If the inner function doesn't reference any outer variable, it's just an inner function, not a closure.
