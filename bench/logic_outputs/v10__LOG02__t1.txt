No — `acc` is a footgun. The default `acc=[]` creates a single shared list across all calls (mutable default argument). Every call without an explicit `acc` mutates the same list. Fix the bug first, then name the parameter:

```python
def collect(item, acc=None):
    if acc is None:
        acc = []
    acc.append(item)
    return acc
```

With the bug fixed, `acc` is fine for a one-liner accumulator. If the function grows, spell it out (`accumulator` or `results`).
