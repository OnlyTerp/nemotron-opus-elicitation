**No.** "Short enough to skip types" is a false assumption — type hints pay off most on small, pure functions because they're the ones you reuse and compose. This function also has an off-by-one bug: `range(len(xs) - k)` drops the last window; it should be `range(len(xs) - k + 1)`.

```python
from collections.abc import Sequence

def moving_average(xs: Sequence[float], k: int) -> list[float]:
    out = []
    for i in range(len(xs) - k + 1):
        window = xs[i:i + k]
        out.append(sum(window) / k)
    return out
```

Type hints catch the `int`/`float` mismatch at the call site; the `+1` fixes the off-by-one. Both take seconds to add.
