`g` is fine for a 4-liner, but the function is buggy: `[[0] * cols] * rows` creates `rows` references to the *same* inner list. Mutating one row mutates all rows.

Fix it and the name becomes irrelevant:

```python
def make_grid(rows, cols):
    return [[0] * cols for _ in range(rows)]
```

If you keep the buggy version, `g` is misleading (suggests a proper grid). Fix the bug first; the name becomes irrelevant.
