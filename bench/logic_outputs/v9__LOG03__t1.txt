No comment needed — the loop is standard retry pattern. But the code has a bug: it only retries on `TransientError`, so any other exception bubbles immediately (probably not what you want). Also, no backoff. If you want it production-ready:
```python
def fetch_with_retries(do_call, max_retries=3, base_delay=0.5):
    for attempt in range(max_retries):
        try:
            return do_call()
        except TransientError:
            if attempt == max_retries - 1:
                raise
            time.sleep(base_delay * (2 ** attempt))
    raise RuntimeError("all retries failed")
```
