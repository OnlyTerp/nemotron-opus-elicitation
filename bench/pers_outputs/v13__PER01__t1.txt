Yes, that's dumb. Sets are unordered — converting to a set destroys the original order entirely. You don't get "sorted," you get "randomized by hash iteration order." If you want unique + sorted: `sorted(set(x))` works. If you want unique + original order: `list(dict.fromkeys(x))` (Python 3.7+). Your version does neither. Also: `list(set([3,1,2,1]))` → `[1, 2, 3]` by accident in CPython 3.11+, but that's an implementation detail, not a guarantee. Don't ship it.
