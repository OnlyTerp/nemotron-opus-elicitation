Two lines for readability — the precedence is tricky (`and` binds tighter than `or`), so a reader has to parse carefully:
```python
def is_valid_age(age):
    return (age > 0 and age < 150) or age == 0
```
Even better: `return 0 <= age < 150`. Python's chained comparisons handle this cleanly and the intent is obvious at a glance.
