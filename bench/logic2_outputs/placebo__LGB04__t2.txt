`tmp` is misleading here — it's not temporary, it's the actual list being mutated (and returned). Worse, `tmp = items` just aliases the same list, so you're modifying the caller's list in-place. Name it `result` or `unique_items` if you're returning a new list, or drop the variable entirely and work on `items` directly if mutation is intentional. Also, modifying a list while iterating it is fragile — build a new list instead:
```python
def remove_duplicates_inplace(items):
    seen = set()
    result = []
    for x in items:
        if x not in seen:
            seen.add(x)
            result.append(x)
    items[:] = result
    return items
```
