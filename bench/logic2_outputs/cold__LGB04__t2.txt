`tmp` is **misleading** here — it's not a temporary copy, it's the **same list object** (`tmp = items` creates another reference, not a copy). So `tmp.remove(x)` mutates the original `items` in place, and `return tmp` returns that same mutated list.

Better names: `result` or `result_list` (if you want to emphasize it's the return value), or actually make a copy: `result = items.copy()` / `result = list(items)` if you want to preserve the original.

Also, the function has a bug: modifying a list while iterating over it (`for x in items: ... tmp.remove(x)`) skips elements. Use a list comprehension or build a new list instead:

```python
def remove_duplicates(items):
    seen = set()
    return [x for x in items if not (x in seen or seen.add(x))]
```
