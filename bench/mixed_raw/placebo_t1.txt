Close! A closure is a function *that remembers the variables from its enclosing scope* — even after that outer function has finished running. The inner function "closes over" those variables, capturing them by reference (in Python) or by value (in some other languages).

```python
def make_multiplier(factor):
    def multiply(x):          # this inner function is a closure
        return x * factor     # it "closes over" `factor`
    return multiply

times3 = make_multiplier(3)
times3(4)  # 12 — `factor` is still accessible!
```

The key distinction: a plain nested function doesn't become a closure unless it actually *uses* variables from outside. And the magic is that those variables stay alive as long as the closure exists. You've got the right intuition — the nesting is the mechanism, but the *capture* is what makes it a closure.

They're equivalent in *computational power* (anything recursion can do, iteration can do, and vice versa), but they're not "basically the same" in practice.

- **Stack usage**: Recursion consumes call stack; deep recursion hits stack overflow. Iteration uses constant stack space.
- **Tail calls**: Some languages eliminate tail calls; Python doesn't.
- **Readability**: Recursion shines for tree/graph traversal; iteration wins for simple loops.

Rule of thumb: use recursion when the problem structure is recursive and depth is bounded; use iteration for simple repetition.

Actually, Python lists *are* arrays of pointers — but indexing is **O(1) everywhere**. The list object holds a contiguous C array of `PyObject*` pointers. `lst[i]` is a single pointer dereference. Middle, end, beginning — all the same speed.

What *is* slow: inserting/deleting at the front or middle (`O(n)` shift). Reading? Constant time. If you're doing millions of index lookups and it's slow, the bottleneck is what you're doing with the element, not the indexing. For fast front inserts, `collections.deque` gives O(1) both ends. But don't optimize indexing — it's already optimal.

```python
def fib(n: int) -> int:
    """Return the nth Fibonacci number (fib(0)=0, fib(1)=1)."""
    a, b = 0, 1
    for _ in range(n):
        a, b = b, a + b
    return a


assert fib(0) == 0
assert fib(10) == 55
```

Iterative, O(n) time, O(1) space. `fib(10)` = 55 is a good sanity check.
