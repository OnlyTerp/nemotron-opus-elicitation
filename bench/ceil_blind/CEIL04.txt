def median(xs):

--- Candidate G ---
The code is **wrong**, not just undocumented.

**Bugs found:**
1. **Empty list** → `IndexError` (no guard)
2. **Even-length lists** → returns upper-middle element, not the median. `[1,2]` returns `2` (should be `1.5`); `[1,2,3,4]` returns `3` (should be `2.5`)

```python
def median(xs):
    """Return the median of a sequence. Raises ValueError if empty."""
    s = sorted(xs)
    n = len(s)
    if n == 0:
        raise ValueError("median() arg is an empty sequence")
    mid = n // 2
    if n % 2 == 0:
        return (s[mid - 1] + s[mid]) / 2
    return s[mid]
```

A docstring is necessary *after* fixing the logic — the current behavior is silently wrong for half the inputs.

--- Candidate H ---
**No — it's wrong, not just undocumented.**

The function crashes on empty lists and returns the upper middle element for even-length lists instead of averaging the two middle values. On `[1,2]` it returns `2` (should be `1.5`); on `[1,2,3,4]` it returns `3` (should be `2.5`). It only works by accident for odd-length inputs.

```python
def median(xs):
    """Return the median of a non-empty sequence."""
    s = sorted(xs)
    n = len(s)
    if n == 0:
        raise ValueError("median of empty sequence")
    mid = n // 2
    if n % 2 == 1:
        return s[mid]
    return (s[mid - 1] + s[mid]) / 2
```
