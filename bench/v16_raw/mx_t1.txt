You've got the nesting right but missed the essential part. A closure is a function *plus* the environment it captured — the non-local variables it references from its enclosing scope. The inner function alone isn't a closure until it "closes over" something.

```python
def make_adder(x):
    def adder(y):
        return x + y
    return adder

add5 = make_adder(5)
add5(3)  # 8
```

`adder` becomes a closure at runtime when `make_adder(5)` returns it, binding `x=5` into `adder.__closure__[0].cell_contents`. A nested function that doesn't reference outer variables has `__closure__ == None`.

They're equivalent in *computational power*, but they're not "basically the same." Recursion uses the call stack; iteration uses a loop variable. That difference matters: deep recursion crashes (Python limit ~1000). Tail calls: Python doesn't optimize them. State: recursion threads state through parameters; iteration mutates locals. Readability: some algorithms (tree traversal) are naturally recursive. Recursion isn't "a loop that calls itself" — it's a function that calls itself. Use iteration by default in Python; recurse when the problem structure demands it.

Wrong. Python lists are arrays of pointers to objects, stored contiguously. Indexing is O(1) — `list[i]` is pointer arithmetic: `base + i * sizeof(void*)`. Middle vs. end makes zero difference. What *is* slow: inserting/deleting at the front or middle (O(n) because everything shifts). If you need fast front operations, use `collections.deque`. Don't optimize indexing — it's already optimal.

```python
def retry(fn, times):

        try:
            return fn()
        except Exception:
            if i == times - 1:
                raise
```

```python
def is_sorted(xs):
    for i in range(len(xs) - 1):
        if xs[i] > xs[i + 1]:
            return False
    return True
```

Or Pythonic: `all(xs[i] <= xs[i+1] for i in range(len(xs)-1))`.

```python
def fib(n: int) -> int:
    a, b = 0, 1
    for _ in range(n):
        a, b = b, a + b
    return a

assert fib(0) == 0
assert fib(1) == 1
assert fib(10) == 55
```

Iterative, O(n) time, O(1) space. Handles `n=0` correctly (loop runs 0 times, returns `a=0`).
