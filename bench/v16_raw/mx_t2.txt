You're half right. A closure *is* a function defined inside another function — but the defining feature is that it **captures and retains references to variables from its enclosing scope**, even after that outer function has finished executing.

```python
def outer(x):
    def inner():
        return x  # captures x
    return inner

f = outer(5)
f()  # returns 5 — x is "closed over"
```

The nested function alone isn't the closure; the *combination* of the function + its captured environment is. Common pitfall — late binding in loops: `funcs = [lambda: i for i in range(3)]` gives `[2, 2, 2]`. Fix: `lambda i=i: i`.

They compute the same things, but they're not "basically the same." Recursion uses the call stack; iteration uses a loop variable. Stack overflow: deep recursion crashes (limit ~1000). Tail calls: Python doesn't optimize them. State: recursion threads state through parameters; iteration mutates locals. Recursion isn't "a loop that calls itself" — it's a function that calls itself. Use iteration by default in Python; recurse when the problem structure demands it.

Wrong. Python lists are arrays of pointers to objects, stored contiguously. Indexing is O(1) — `list[i]` is pointer arithmetic: `base + i * sizeof(void*)`. Middle vs. end makes zero difference. What *is* slow: inserting/deleting at the front or middle (O(n) because everything shifts). If you need fast front operations, use `collections.deque`. Don't optimize indexing — it's already optimal.

`fn` is fine for a tiny callback — it's a well-known convention (Haskell, Rust, JS). But your function has a bug: on the last iteration after all retries fail, you call `fn()` again without try/except, so the final exception propagates. That's probably not what you want.

```python
def retry(fn, times):
    for i in range(times):
        try:
            return fn()
        except Exception:
            if i == times - 1:
                raise
```

def running_max(nums: list[int]) -> int | None:
    if not nums:
        return None
    total = nums[0]
    for n in nums:
        if n > total:
            total = n
    return total

```python
def fib(n: int) -> int:
    if n < 2:
        return n
    a, b = 0, 1
    for _ in range(n - 1):
        a, b = b, a + b
    return b

assert fib(0) == 0
assert fib(1) == 1
assert fib(10) == 55
```

Iterative, O(n) time, O(1) space. Handles `n=0` and `n=1` without special-casing the loop.
