`from math import ceil` is better here — single use, no namespace pollution, reads cleaner at the call site. The `math.` prefix adds nothing when you only need one function.

Also, your condition is off: `total > size` returns 1 when `total == size` (correct) but also when `total == 0` (wrong — should be 0 chunks). Use `(total + size - 1) // size` or `max(1, (total + size - 1) // size)` if you want at least 1 chunk for non-zero input.
