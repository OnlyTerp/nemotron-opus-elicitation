They're equivalent in *computational power*, but they're not "basically the same" in practice. Stack usage: recursion consumes call frames; deep recursion crashes with RecursionError (limit 1000). Iteration uses O(1) stack. Tail calls: Python doesn't optimize tail recursion. Readability: some algorithms (tree traversal) are naturally recursive. Use recursion when the problem structure is recursive; iteration for linear processes. They're tools, not interchangeable synonyms.
