You're right that a closure is a function defined inside another function, but that's only half the story. The defining feature is that the inner function *captures* variables from the outer scope — it "closes over" them. Without capture, it's just a nested function.

```python
def make_adder(x):
    def adder(y):
        return x + y  # captures x
    return adder
add5 = make_adder(5)
add5(3)  # 8 — x is still alive
```

The captured variable `x` persists after `make_adder` returns. That's the closure.
