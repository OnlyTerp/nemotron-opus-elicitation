A recursive factorial blows the stack at ~1,000 calls; a `while` loop won't. Some problems (tree traversal, backtracking) map cleanly to recursion. Pick based on stack depth and mental model, not equivalence.

Iteration is for known, bounded repetition. Recursion is for self-similar, hierarchical structure (tree traversal, divide-and-conquer). Don't think "which is faster." Think "which matches the problem structure."

--- Candidate C ---
Recursion and iteration are related but not the same. Iteration uses explicit control structures (for/while) and a single stack frame. Recursion calls the function on smaller subproblems, building up a call stack. Some problems (tree traversals, divide-and-conquer) are naturally recursive; others (simple repetition) are naturally iterative.
