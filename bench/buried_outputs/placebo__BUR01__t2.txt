You're asking the wrong question. `cudaDeviceSynchronize()` between kernels on the same stream is redundant — the stream already serializes them. For a memory-bound kernel, occupancy isn't your limiter; memory bandwidth is. Block size barely matters once you have enough blocks to fill SMs (typically 2–4 blocks/SM). The real win: fuse kernels, use async copy (TMA/cudaMemcpyAsync), or restructure to reduce global memory traffic. If you must tune block size, profile with NCU — try 256 first, it's the sweet spot for most architectures.
