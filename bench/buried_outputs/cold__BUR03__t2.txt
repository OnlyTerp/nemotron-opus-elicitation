**Fix the connection overhead first — partitioning won't help if you're reconnecting per event.**

```python
# Current: ~1-2ms per connect + 0.5ms INSERT = terrible throughput
# Fix: connection pooling + batching

from psycopg2.pool import ThreadedConnectionPool
from psycopg2.extras import execute_batch

pool = ThreadedConnectionPool(1, 20, dsn="postgresql://...")

def write_events(events):
    with pool.getconn() as conn:
        with conn.cursor() as cur:
            execute_batch(cur, 
                "INSERT INTO events (type, payload, ts) VALUES (%s, %s, %s)",
                [(e.type, e.payload, e.ts) for e in events],
                page_size=1000)
        conn.commit()
```

**On partitioning:** Only consider if you hit *actual* write contention (lock waits, not connection overhead). For time-series events:
- Partition by time (monthly/weekly) via `pg_partman` or native declarative partitioning
- 1 partition per time bucket keeps indexes small and enables fast `DROP PARTITION` for retention
- Start with monthly; adjust based on data volume and query patterns

**Expected gain:** Pool + batch = 10-100× throughput. Partitioning = marginal unless you have billions of rows and partition pruning helps reads.</think>
