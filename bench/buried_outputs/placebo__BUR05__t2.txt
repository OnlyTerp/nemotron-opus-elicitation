Creating a new `sync.Mutex` per request does nothing — each goroutine gets its own mutex, so there's zero contention protection. The mutex must be shared (package-level or on a shared struct). Fix that first. Then: `GOMAXPROCS` defaults to CPU cores — don't touch it unless you have a specific reason. Worker pool size? Start with `runtime.NumCPU()` and tune via load testing. But your real throughput killer is likely lock contention or blocking I/O — profile with `pprof` before tuning knobs.
