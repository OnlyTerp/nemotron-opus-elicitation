**Fix the Dataset first — num_workers/pin_memory are irrelevant when you reload 8GB per sample.**

```python
# Current: 8GB read × batch_size × epochs = impossible
# Fix: load once, index efficiently

class CSVDataset(Dataset):
    def __init__(self, csv_path):
        # Option 1: Memory-map (if fits in RAM)
        self.df = pd.read_csv(csv_path)  # Load ONCE
        
        # Option 2: If too large, use chunked/parquet
        # self.df = pd.read_parquet(csv_path.replace('.csv', '.parquet'))
        
    def __len__(self):
        return len(self.df)
    
    def __getitem__(self, idx):
        row = self.df.iloc[idx]  # O(1) index
        return torch.tensor(row.values), torch.tensor(row.target)
```

**Then tune DataLoader:**
```python
loader = DataLoader(dataset, batch_size=256, num_workers=4, 
                    pin_memory=True, persistent_workers=True)
```
- `num_workers=4-8` (CPU cores / 2) — only helps if __getitem__ does real work (decode, augment)
- `pin_memory=True` — speeds GPU transfer, enable if using GPU
- `persistent_workers=True` — avoids worker restart overhead per epoch

**Real fix:** Convert CSV → Parquet once (`df.to_parquet()`), then `read_parquet` is 10-50× faster with column pruning.
