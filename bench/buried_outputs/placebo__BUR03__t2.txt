Partitions won't help — your bottleneck is `psycopg2.connect` per event. That's ~1–5 ms handshake per INSERT. Use a connection pool:

```python
from psycopg2.pool import ThreadedConnectionPool
pool = ThreadedConnectionPool(1, 20, dsn=...)
conn = pool.getconn()
try:
    with conn.cursor() as cur:
        cur.execute("INSERT ...", event)
    conn.commit()
finally:
    pool.putconn(conn)
```

Batch inserts (multi-row `VALUES` or `COPY`) for 10–100× throughput. Partitions only matter if you're hitting a single partition's lock contention at massive scale — fix the connection pattern first.
