Creating a new `sync.Mutex` per request provides zero synchronization — each request gets its own mutex, so there's no contention to protect. Remove it. `GOMAXPROCS` defaults to CPU cores (correct); only change if you're cgroup-limited. For throughput: profile first (`pprof`). Likely fixes: connection pooling, batching, async I/O, or reducing lock scope if you have *shared* state. A worker pool only helps if you're doing CPU-bound work per request — otherwise it adds overhead.
