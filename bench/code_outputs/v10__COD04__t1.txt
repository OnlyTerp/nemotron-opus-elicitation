1024 bytes is too small — you'll make ~1M syscalls per GB. Use 64 KiB (65536) or 128 KiB; modern disks and OS page caches handle larger sequential reads efficiently. Also, the function leaks file handles — use `with open(...) as f:` for both files. And `shutil.copyfileobj(f, g, 65536)` does this correctly in one line. If you must write it manually: `CHUNK = 64 * 1024` and wrap in `with` statements.
