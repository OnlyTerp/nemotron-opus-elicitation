Uppercase is the common convention (`SELECT * FROM users WHERE name = ?`), but **your code has a SQL injection vulnerability** — never interpolate variables into queries.

Use parameterized queries:
```python
def get_user(conn, username):
    q = "SELECT * FROM users WHERE name = ?"
    return conn.execute(q, (username,)).fetchone()
```

The `?` placeholder (or `%s` for psycopg2, `:name` for named params) lets the driver handle escaping safely. Casing is a style choice; parameterization is not optional.
