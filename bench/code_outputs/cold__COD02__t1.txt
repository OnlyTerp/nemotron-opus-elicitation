Uppercase for SQL keywords is the more common convention in Python codebases — it visually separates the query structure from Python code and column/table identifiers. Most style guides (e.g., Django, SQLAlchemy examples) use `SELECT`, `FROM`, `WHERE`.

That said, consistency matters more than the specific choice. Pick one and enforce it via linter/formatter (e.g., `sqlfluff` with a rule config).

**Bigger issue:** your snippet has a SQL injection vulnerability — use parameterized queries:

```python
def get_user(conn, username):
    q = "SELECT * FROM users WHERE name = ?"
    return conn.execute(q, (username,)).fetchone()
```

(Use `%s` for psycopg2, `?` for sqlite3, `:name` for Oracle — never string interpolation.)
