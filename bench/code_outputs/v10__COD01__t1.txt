`r` is fine here — it's a conventional name for "result" or "response" in a tight loop, and the scope is ~10 lines. The real bug: `lo = mid` on line 17 should be `lo = mid + 1`, otherwise the loop never terminates when `target > a[mid]` (you keep re-testing the same `mid`). Same for `hi = mid` on line 19 — should be `hi = mid - 1` for standard binary search, though `hi = mid` works if you keep `while lo < hi` and return `lo` at the end. But as written, it infinite-loops.
