**Uppercase** is the more common convention in SQL — it visually separates keywords from identifiers, making queries easier to scan. Most style guides (Google, Django, SQLAlchemy) recommend it.

More importantly, this code has a **SQL injection vulnerability**. Never interpolate values directly into queries. Use parameterized queries:

```python
def get_user(conn, username):
    q = "SELECT * FROM users WHERE name = ?"
    return conn.execute(q, (username,)).fetchone()
```

The `?` placeholder (or `%s` for psycopg2, `%s`/`?` depending on driver) lets the database handle escaping safely. This is non-negotiable for any code touching user input.
