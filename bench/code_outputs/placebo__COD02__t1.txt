Uppercase is the traditional SQL convention (`SELECT * FROM users WHERE name = ?`) and reads better in mixed code, but lowercase works fine if your team agrees. Pick one and be consistent. More importantly: that string interpolation (`'%s' % username`) is a SQL injection vulnerability. Use parameterized queries: `q = "SELECT * FROM users WHERE name = %s"` then `conn.execute(q, (username,))`. The case style is bikeshedding; the injection hole is the real issue.
