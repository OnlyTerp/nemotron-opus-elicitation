You're right that a closure is a function defined inside another function, but that's only half the story. The defining feature is that the inner function *captures* variables from the outer scope — it "closes over" them. Without capture, it's just a nested function.

```python
def make_adder(x):
    def adder(y):
        return x + y  # captures x
    return adder
add5 = make_adder(5)
add5(3)  # 8 — x is still alive
```

The captured variable `x` persists after `make_adder` returns. That's the closure.

They compute the same things, but they're not "basically the same." Recursion uses the call stack; iteration uses a loop variable. Stack overflow: deep recursion crashes (limit ~1000). Tail calls: Python doesn't optimize them. State: recursion threads state through parameters; iteration mutates locals. Recursion isn't "a loop that calls itself" — it's a function that calls itself. Use iteration by default in Python; recurse when the problem structure demands it.

```python
def fib(n: int) -> int:
    if n < 2:
        return n
    a, b = 0, 1
    for _ in range(n - 1):
        a, b = b, a + b
    return b

assert fib(0) == 0
assert fib(1) == 1
assert fib(10) == 55
```
Iterative, O(n) time, O(1) space.
