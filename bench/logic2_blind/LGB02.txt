USER MESSAGE:
Does the inner loop need a comment, or is it clear?
```python
def group_sums(rows):
    result = []
    total = 0
    for row in rows:
        for x in row:
            total += x
        result.append(total)
    return result
```

CANDIDATE ANSWERS:

--- Candidate A ---
The inner loop is clear — it's a simple sum accumulation. No comment needed. The logic is obvious: `total += x` for each element. Comments that restate code add noise.

--- Candidate B ---
The bug is louder than the comment question: `total` accumulates across rows instead of resetting per row. Fix that first — `total = 0` inside the outer loop. After that, the logic is self-explanatory; no comment needed.

--- Candidate C ---
The inner loop itself is clear — `for x in row: total += x` is an idiomatic sum. **But the function is buggy:** `total` isn't reset per row, so it returns a cumulative sum across all rows, not per-row sums. Fix that first:

```python
def group_sums(rows):
    result = []
    for row in rows:
        total = 0
        for x in row:
            total += x
        result.append(total)
    return result
```

Or more idiomatically: `return [sum(row) for row in rows]`.

With the fix, the inner loop (or `sum()`) is self-explanatory — no comment needed. If you keep the explicit loop, a comment would only restate the obvious.
