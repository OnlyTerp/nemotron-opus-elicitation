USER MESSAGE:
Rename `c` to `count`, or is `c` fine?
```python
def average(nums):
    s = 0
    c = 0
    for n in nums:
        s += n
        c += 1
    return s // c
```

CANDIDATE ANSWERS:

--- Candidate A ---
Rename to `count`. `c` is ambiguous (character? char code? constant?) and saves 4 chars at the cost of readability. `s` → `total` for the same reason. The whole function is `sum(nums) // len(nums)` if you're not avoiding builtins.

--- Candidate B ---
Rename to `count`. `c` is too cryptic for a loop counter — adds cognitive load for zero savings. `count` is clear, conventional, and same length. Also consider `sum(nums) // len(nums)` instead of manual loop.

--- Candidate C ---
Rename to `count`. Single-letter names are fine for tight loop indices (`i`, `j`) or very short scopes, but `c` accumulates across the whole function — it's a meaningful variable, not a throwaway. `count` (or `n` if you want mathematical brevity) costs nothing and removes ambiguity. Same for `s` → `total`. The function becomes self-documenting:

```python
def average(nums):
    total = 0
    count = 0
    for n in nums:
        total += n
        count += 1
    return total // count
```
