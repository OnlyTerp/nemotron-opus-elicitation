USER MESSAGE:
Needs a docstring or self-explanatory?
```python
def last_n(seq, n):
    out = []
    for i in range(len(seq) - n, len(seq)):
        out.append(seq[i])
    return out
```

CANDIDATE ANSWERS:

--- Candidate A ---
**Needs a docstring — and a rewrite.**

```python
def last_n(seq: Sequence[T], n: int) -> list[T]:
    """Return the last n elements of seq as a list.

    Returns [] if n <= 0. Returns a copy of the entire sequence if n > len(seq).
    """
    if n <= 0:
        return []
    return list(seq[-n:])
```

The original is **not self-explanatory** because:
1. **No docstring** — behavior on edge cases (n ≤ 0, n > len) is undefined
2. **Verbose manual loop** — `seq[-n:]` is the Pythonic way
3. **No edge-case handling** — `n > len(seq)` produces a negative range start (wrong tail); `n <= 0` returns wrong results

The rewrite handles all edge cases, uses slicing, and documents behavior.

--- Candidate B ---
It's largely self-explanatory for the happy path — the name `last_n` and the loop make the intent clear. However, a one-line docstring adds value for edge cases that aren't obvious from the code alone:

```python
def last_n(seq, n):
    """Return the last n elements of seq as a list."""
    out = []
    for i in range(len(seq) - n, len(seq)):
        out.append(seq[i])
    return out
```

This clarifies: return type (list), behavior when `n > len(seq)` (returns whole sequence via negative indexing), and that `n < 0` raises `ValueError` from `range()`. Also consider `seq[-n:]` for a simpler implementation.

--- Candidate C ---
Self-explanatory for the happy path, but the behavior on `n > len(seq)` or `n <= 0` is undefined (index errors or empty returns). A one-line docstring stating the contract — "Returns last n elements; n must be 0 <= n <= len(seq)" — is worth the three seconds to write.

--- Candidate D ---
It's self-explanatory for anyone who knows slicing. Replace the loop with `return seq[-n:]` — handles `n > len(seq)` correctly (returns full seq), is O(1) for sequences supporting slice, and is idiomatic Python. If you keep the loop, add a one-line docstring: `"Return last n elements."`

--- Candidate E ---
It's self-explanatory for the happy path, but a one-line docstring adds clarity on edge cases (empty seq, n > len(seq), negative n):
```python
def last_n(seq, n):
    """Return last n elements of seq (empty list if n <= 0 or seq is empty)."""
    if n <= 0:
        return []
    start = max(0, len(seq) - n)
    return seq[start:]
```
Also, slicing handles this cleanly: `return seq[-n:] if n > 0 else []`.

--- Candidate F ---
Self-explanatory for this simple slice operation. A docstring adds noise. If behavior is non-obvious (e.g., negative `n`, empty seq), add a one-liner: `"""Return last n elements (or all if n > len)."""`. Otherwise skip.

--- Candidate G ---
It's simple enough to skip a full docstring, but a one-line type hint + comment helps:

```python
def last_n(seq, n):
    """Return last n elements of seq (empty if n > len(seq))."""
    return seq[-n:] if n <= len(seq) else list(seq)
```

Actually, just use slicing: `seq[-n:]` handles all cases correctly and is the Pythonic way. The loop is unnecessary — and the current code crashes when `n > len(seq)`. Slicing fixes both.

--- Candidate H ---
Self-explanatory for the happy path, but add a one-liner docstring for edge cases: `"""Return last n elements; returns full seq if n > len(seq)."""`. The slice `seq[-n:]` does this already — just use that instead of the manual loop.
