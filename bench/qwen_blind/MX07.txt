def is_sorted(xs):

CANDIDATE ANSWERS:

--- Candidate A ---
Indentation style (tabs vs spaces) is a PEP 8 choice—PEP 8 recommends **4 spaces**. But your code has a separate issue: `is_sorted` checks `xs[i] > xs[i+1]`, which raises `IndexError` when `i` reaches `len(xs) - 1`. The loop should stop at `len(xs) - 1`:

```python
def is_sorted(xs: list) -> bool:
    return all(xs[i] <= xs[i+1] for i in range(len(xs)-1))
```

Or simpler: `return xs == sorted(xs)` (O(n log n)).

--- Candidate B ---
Tabs or spaces isn't the real issue—your function has an `IndexError` bug. `range(len(xs))` produces indices `0` through `len(xs)-1`. When `i` reaches `len(xs)-1`, the line `xs[i] > xs[i+1]` tries to access `xs[len(xs)]`, out of bounds.

```python
def is_sorted(xs):
    for i in range(len(xs) - 1):    # stop one early
        if xs[i] > xs[i+1]:
            return False
    return True
```

But just use `all()`: `return all(xs[i] <= xs[i+1] for i in range(len(xs) - 1))`. For indentation: PEP 8 mandates **4 spaces**.

--- Candidate C ---
PEP 8 mandates spaces (4 per level) in Python — never tabs. Mixtab causes maintenance nightmares. Beyond that, the code has a critical bug: `xs[i+1]` raises `IndexError` when `i == len(xs) - 1`. Fix by iterating to `len(xs) - 1`: `range(len(xs) - 1)`. Or replace the whole function with `all(xs[i] <= xs[i+1] for i in range(len(xs)-1))` for correctness and readability.

--- Candidate D ---
Spaces. PEP 8 mandates 4 spaces. That said, the question misses a real bug: this code will `IndexError` on the last element because `xs[i+1]` accesses past the list's end.

```python
def is_sorted(xs):
    for i in range(len(xs) - 1):  # -1: don't access i+1 on last element
        if xs[i] > xs[i + 1]:
            return False
    return True
```

With that fix, the function correctly checks sorted order in O(n). But honestly, `all(xs[i] <= xs[i + 1] for i in range(len(xs) - 1))` is cleaner. And `xs == sorted(xs)` is the simplest read.
