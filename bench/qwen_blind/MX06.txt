USER MESSAGE:
Should I add a type hint to `total`?
```python
def running_max(nums):
    total = nums[0]
    for n in nums:
        if n > total:
            total = total
    return total
```

CANDIDATE ANSWERS:

--- Candidate A ---
The variable is misnamed and the function has a bug that type hints won't fix. `running_max` suggests it tracks the maximum, but `total = total` is a no-op. The function just returns `nums[0]`.

```python
def running_max(nums: list[float]) -> float:
    current_max = nums[0]
    for n in nums:
        if n > current_max:
            current_max = n
    return current_max
```

Yes, adding `float` type hints is fine, but first fix the bug. Also: Python already has `max(nums)` which does this in C.

--- Candidate B ---
The bigger issue isn't type hints — the function is buggy. `total = total` is a no-op, so the variable never updates. If the intent is running max, it should be `total = n`. (Also, the function name `running_max` with `total = nums[0]` suggests it might be computing something else — clarify the intent.) Fix the logic first. As for type hints, adding `total: float = nums[0]` is good practice for clarity, but won't fix the no-op assignment.

--- Candidate C ---
Type-hinting `total` isn't the problem here—rewriting the function is. The function as written doesn't compute a running maximum. It assigns `total = total` in the loop body, which is a no-op. It just returns `nums[0]`.

```python
def running_max(nums):
    total = nums[0]
    for n in nums:
        if n > total:
            total = total    # ← no-op, should be: total = n
    return total
```

Fix that first. With the correction, you *could* just use `max(nums)`, which is implemented in C. Type hints on locals are almost never necessary—type checkers infer them from the right-hand side.

--- Candidate D ---
Yes — but the code has a real bug, not a typing question. Line 11 does `total = total`, which is a no-op. The function is supposed to compute the running max, but it never updates `total`. It just returns `nums[0]` every time.

```python
def running_max(nums: list[int]) -> int:
    total = nums[0]
    for n in nums[1:]:
        if n > total:
            total = n  # was: total = total (no-op)
    return total
```

Fixes: (1) `total = n` instead of `total = total`. (2) Skip the first element with `nums[1:]`. (3) Edge case: `nums[0]` raises IndexError on empty list — guard that first.
