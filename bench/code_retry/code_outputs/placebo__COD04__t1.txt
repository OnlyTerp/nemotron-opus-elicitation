**1024 is too small** for modern systems. It'll work but causes excessive syscall overhead (millions of `read`/`write` calls per GB).

**Sensible defaults:**
- **64 KiB (65536)** — sweet spot for most workloads; matches typical kernel readahead, fits in L1/L2 cache
- **128–256 KiB** — better for SSDs/NVMe, reduces syscall frequency further
- **1 MiB** — fine for large sequential copies on fast storage

**Practical pattern:** Use `shutil.copyfileobj(f, g, length=65536)` — it's optimized in C, handles edge cases, and lets callers override.

**Other issues in your snippet:**
- No `with` statements — files won't close on exception
- No error handling
- `shutil.copy2(src, dst)` or `copyfile(src, dst)` is preferred for most cases (preserves metadata, uses `sendfile`/`copy_file_range` on Linux)

If you're rolling your own for learning: bump to 64 KiB, add `with`, and you're good.
