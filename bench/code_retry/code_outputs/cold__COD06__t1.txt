This code absolutely needs a docstring — and more critically, it has serious bugs.

**Race condition:** Multiple threads increment `counter += 1` without synchronization. This is not atomic in Python (GIL doesn't help here — the bytecode `INPLACE_ADD` + `STORE_GLOBAL` can interleave). You'll lose increments.

**Lambda capture bug:** `lambda: add(n)` captures `n` by reference. If `n` changes before threads run, they'll use the wrong value. Use `lambda n=n: add(n)` or `functools.partial`.

**Missing import:** `threading` is used but not imported.

**Global state:** Mutable globals make this hard to test and reason about. Consider a class with a `Lock` or `threading.local`/`concurrent.futures`.

A docstring should at minimum document: what `n` means, that it spawns 8 threads, the race condition caveat, and that `counter` is global state. But really, fix the bugs first — documenting broken code is worse than no docs.
