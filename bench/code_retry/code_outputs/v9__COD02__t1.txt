Uppercase for SQL keywords — it's the de facto standard in the SQL world (PostgreSQL, MySQL, SQLite, Oracle docs all use it). It also lets you scan queries faster: keywords pop, identifiers don't. Your real problem here is the `%s` interpolation — that's a SQL injection hole. Use parameterized queries: `q = "SELECT * FROM users WHERE name = ?"` then `conn.execute(q, (username,))`.
