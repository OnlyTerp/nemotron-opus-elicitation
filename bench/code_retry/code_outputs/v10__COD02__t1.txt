**Uppercase is the common convention** (SELECT, FROM, WHERE) — it visually distinguishes keywords from identifiers. But that's the least of your problems.

**Critical flaw:** `"select * from users where name = '%s'" % username` is a **SQL injection vulnerability**. Never interpolate values into SQL strings.

**Fix:** Use parameterized queries:
```python
def get_user(conn, username):
    q = "SELECT * FROM users WHERE name = ?"
    return conn.execute(q, (username,)).fetchone()
```

The `?` placeholder (or `%s` for psycopg2, `:name` for named params) lets the driver handle escaping safely. The database parses the query plan once, then binds values — also faster for repeated calls.

Style guide entry: "SQL keywords UPPERCASE; always use parameterized queries; never string-interpolate user input."
