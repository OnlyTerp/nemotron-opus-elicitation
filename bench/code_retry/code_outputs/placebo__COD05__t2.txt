This has a data race on `counter` — eight threads doing `counter += 1` without synchronization will lose increments. `+=` isn't atomic in Python (GIL doesn't help here; it's bytecode-level). Use `threading.Lock` or `concurrent.futures.ThreadPoolExecutor` with a shared `queue.Queue`, or better: `counter = sum(futures)` via `executor.map`. Also, `getData` should be `fetch_json` or `fetch_all_json` — `getData` says nothing about what it returns.
